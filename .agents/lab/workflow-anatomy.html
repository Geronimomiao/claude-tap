SEC.01

Ultracode 在坐标系里的位置

Effort 档位控制单个模型的思考深度；滑杆顶端其实是一个分叉——同样的预算，两种花法。Ultracode 严格说不是模型档位，而是 harness 的 setting：发给模型的仍是 xhigh，编排行为是 Claude Code 这一层加上去的。

low medium high xhigh ⑂ 分叉

MAX

一个脑子想到极限：最深推理、token 花费不设上限。官方文档：「可能边际收益递减、容易 overthinking，建议先测试再广泛采用」。Session-only。

深度极限 → 单体押注

ULTRACODE = xhigh + workflows

每个脑子想得足够深（xhigh），把余量花在结构上：多 agent 并行覆盖 + 对抗验证 + 评审团。质量增益来自交叉核对，而非单体思考的最后一格。Session-only。

深度够用 + 广度拉满 → 结构押注
为什么不配 max？workflow 一次最多派 1,000 个子代理、并发 16——「无上限 + 易 overthink」的档位乘以几百份会失控。而脚本里每个 agent() 可以单独指定 effort：本次 run 全部用了 'low'（机械性摸底），最难的裁决阶段才值得升 max。全局 xhigh、刀刃上按需，比全局 max 精细。

journal.jsonl 是 resume 账本：每个 agent() 调用按 prompt 内容哈希记 started/result。改脚本重跑时，哈希没变的步骤直接回放缓存结果。

SEC.03

脚本本体（即兴写成，已落盘）

这就是"即兴发挥"的实物：为这个任务现写的纯 JavaScript，runtime 在隔离环境里执行。无文件系统、无 Date.now()/Math.random()（会破坏 resume 的确定性）。

lab-map-wf_b6a72ee5-7aa.js …/e8f2bfaf-…c6e0/workflows/scripts/ 
export const meta = {                                      // ← 纯字面量，不许计算1
  name: 'lab-map',
  description: 'Map the claude-ai lab directory…',
  phases: [{ title: 'Inventory' }, { title: 'Summarize' }],
}

phase('Inventory')                                           // 进度分组2
const inv = await agent(
  `List the top-level directories in /Users/simiao/Desktop/claude-ai…
   Pick up to 5 subprojects. Do not deep-read files.`,
  {
    label: 'inventory',
    effort: 'low',                                          // 机械活，便宜档3
    schema: { type: 'object', required: ['items'], … },     // 强制结构化返回4
  },
)

log(`Inventory picked ${inv.items.length} subprojects`)      // 用户可见的进度线

phase('Summarize')
const summaries = await pipeline(inv.items.slice(0, 5), (item) =>  5
  agent(
    `Look inside …/${item.dir}. Skim: read a README or entry file,
     sample at most 2-3 small files. Verify or correct: "${item.guess}".`,
    { label: item.dir, phase: 'Summarize', effort: 'low', schema: … },
  ),
)

return {                                                     // 只有这个回到主会话6
  inventory: inv.items,
  subprojects: summaries.filter(Boolean),                    // 被跳过/挂掉的 agent 是 null
}
  • 1meta 必须是纯字面量——harness 在执行前就要读出名字、描述、phase 列表，用于审批弹窗和进度 UI。
  • 2phase() 只管展示——把后续 agent 归到进度树的同一个组里，方便在 /workflows 里围观。
  • 3effort 按步定价——摸底用 low，若有"裁决哪个发现是真 bug"的阶段才值得 xhigh/max。
  • 4schema 换来免解析——返回不合法 JSON 会在工具层自动重试，脚本永远拿到干净对象。
  • 5pipeline 无 barrier——5 个摸底 agent 同时起跑，谁先完谁先回，墙钟时间 = 最慢一条链。
  • 6中间结果不进上下文——18 万 token 的过程量留在脚本变量里，主会话只收 return 的那一小包。

晋升路径 · 从即兴到命令

  1. 现在：一次性但可追溯。脚本可读、可 diff、可手改后 resume（缓存命中未变步骤）。
  2. 保存：CLI 里 /workflows 选中按 s，或直接让 Claude 存——写入项目 .claude/workflows/（随 repo 分发）或 ~/.claude/workflows/（个人全局）。
  3. 之后：变成 /lab-map 斜杠命令，可带 args 参数化调用。开源分享 = 把它 commit 进仓库。
