你的 prompt 只是原料

「那个新的内容是在哪补充的？」——答案：在你本机的 claude 进程里、请求发出之前。 同一句话被客户端包装成了两种完全不同的请求体。左边是主循环 #2 收到的版本，右边是标题调用 #1 收到的版本。 API 只是被动接收成品；claude-tap 抓到的就是成品。
